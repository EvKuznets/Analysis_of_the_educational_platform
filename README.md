## Описание:

Проект направлен на анализ данных образовательной платформы, которая предлагает студентам пройти курсы по модели trial. Студенты могут бесплатно решать до 30 задач в день, а для неограниченного доступа к задачам в определенной дисциплине необходимо приобрести полный доступ. Команда провела эксперимент с новым экраном оплаты, и теперь необходимо проанализировать данные для получения ценных инсайтов.

## Цель проекта:

Написать оптимальные SQL-запросы, которые предоставят информацию о количестве очень трудолюбивых студентов, а также ключевые метрики для оптимизации воронки: ARPU, ARPAU, CR в покупку, CR активного пользователя в покупку, CR пользователя из активности по математике в покупку курса по математике.

## Данные:

- Трудолюбивый студент: Студент, правильно решивший 20 задач за текущий месяц.
- Активный пользователь: Пользователь, решивший больше 10 задач правильно за все время в любых дисциплинах.
- Активный по математике: Пользователь, за все время решивший 2 или больше задач правильно по математике.

peas – нформация о попытках учеников решить задачи
| Название атрибута | Тип атрибута | Смысловое значение     |
|-------------------|--------------|------------------------|
| st_id             | int          | ID ученика             |
| timest            | timestamp    | Время решения карточки |
| correct           | bool         | Правильное ли решение  |
| subject           | text         | Дисциплина             |

studs – информация об учащихся, участвующих в эксперименте
| Название атрибута | Тип атрибута | Смысловое значение     |
|-------------------|--------------|------------------------|
| st_id             | int          | ID ученика             |
| test_grp          | text         | Метка ученика в данном эксперименте |

final_project_check - информация о покупках, совершенных учениками
| Название атрибута | Тип атрибута | Смысловое значение     |
|-------------------|--------------|------------------------|
| st_id             | int          | ID ученика             |
| sale_time         | timestamp    | Время покупки          |
| money             | int          | Цена покупки курса     |
| subject           | text         | Дисциплина             |

## Результаты проекта:

1. Реализован оптимальный SQL-запрос, который предоставляет информацию о количестве очень трудолюбивых студентов
2. В одном запросе выгружена следующая информация о группах пользователей: ARPU, APRAU, CR в покупку, CR активного пользователя в покупку, CR пользователя из активности по математике в покупку курса по математике.
